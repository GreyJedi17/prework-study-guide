Git

  • Enter your Git notes here
  • git status: checks what branch is being worked currently
  • git checkout -b (branch name): creates a new branch with the given name and switches to it
  • git checkout main: returns to the main branch
  • git restore (file name): restores the branch to originial??? this is not yet confirmed.
  • git clone (SSH key from GitHub): clones the repo from GitHub to local enviornment.
  • git commit : sends local changes to GitHub to be pulled/merged. See below.
  • git flow -- create new branch for local work, make changes (save and commit often), push them to git, submit pull request, team/self merges them to main branch.
    • git add -A: adds my working code to the git staging area
    • git commit -m "note about what was added": commits the code with the added summary.
    • git pull origin main: checks if the main branch is updated.
    • git push origin (branch name): pushes the new branch to GitHub repo to be pulled/merged.
